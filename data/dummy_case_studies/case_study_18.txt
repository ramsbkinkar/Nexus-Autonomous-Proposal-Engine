Project Name: Project Stratus
Industry: Healthcare / Telehealth
The Client Challenge: A regional healthcare provider's monolithic patient portal was originally designed for viewing lab results, not video consultations. When telehealth demand surged, the tightly coupled on-premise infrastructure couldn't scale. The system suffered from frequent database deadlocks, causing patients to drop out of virtual waiting rooms and doctors to lose access to critical electronic medical records (EMR) mid-appointment.
Our Technical Solution: We executed a phased strangler-fig migration to decouple the telehealth features from the legacy monolith. We moved the core application workloads to AWS, containerizing the backend using Docker and orchestrating it via Amazon Elastic Kubernetes Service (EKS). The new patient portal frontend was developed using React, ensuring a responsive, accessible experience across all devices.
For the backend database, we migrated their critical patient scheduling and demographic data from an aging on-premise Microsoft SQL Server to Amazon RDS for PostgreSQL. We implemented strict read-replica routing to ensure that heavy read operations (like doctors pulling up patient histories) did not impact the write operations (like patients booking new appointments).
To handle the real-time video signaling and chat functionality, we built a dedicated microservice using Node.js and WebSockets (Socket.io). This service was entirely stateless, utilizing an ElastiCache (Redis) cluster to manage active session states, allowing the Kubernetes cluster to auto-scale the video-routing pods horizontally during peak morning appointment hours without dropping active connections.
The ROI & Impact: Achieved 99.99% uptime during peak telehealth hours. Reduced average system latency by 65%, allowing doctors to see an average of 3 additional patients per day.
Total Project Cost: $360,000
